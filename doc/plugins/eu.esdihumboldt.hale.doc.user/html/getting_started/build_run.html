Building and running from source Code

If you would like to build HALE from the source code, you need to have the following environment set up:
  • an installation of a 1.6 JDK, which can be found here: http://java.sun.com/javase/downloads/index.jsp
  • an installation of Eclipse for RCP Development, version 3.5, which can be found here:
    http://www.eclipse.org/downloads/download.php? file=/technology/epp/downloads/release/gan ymede/SR2/eclipse-rcp-ganymede-SR2- win32.zip
Furthermore, you need to have a Subersion client installed, such as Subversive or Subclipse for Eclipe, TortoiseSVN or any other one. When these prerequisities are fulfilled, proceed as follows to check out any HALE version later than 1.0.0-RC1:
  1. First, start eclipse in a new workspace.
  2. Second, check out the target platform project for HALE from this location: http://svn.esdi-humboldt.eu/repo/humboldt2/trunk/hale/hale-platform/
  3. After checking out, double click the haleplatform. target file and wait until eclipse has fully resolved all dependencies. This can take a while since some of the dependencies have to be downloaded. When this is finished, press the button labelled "Set as target platform" in the upper right corner.
  4. Now, check out the actual HALE and CST modules as projects:
    1. eu.esdihumboldt.hale: http://svn.esdihumboldt. eu/repo/humboldt2/trunk/hal e/eu.esdihumboldt.hale/
    2. eu.esdihumboldt.hale.gmlparser: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/eu.esdihumboldt. hale.gmlparser/
    3. eu.esdihumboldt.hale.logback.config: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/eu.esdihumboldt. hale.logback.config/
    4. eu.esdihumboldt.hale.rcp.wizards.- functions.core: http://svn.esdi-humboldt. eu/repo/humboldt2/trunk/hale/eu- .esdihumboldt.hale.rcp.wizards.functions. core/
    5. eu.esdihumboldt.hale.schemaprovider: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/eu.esdihumboldt. hale.schemaprovider/
    6. eu.esdihumboldt.hale.htmlexporter: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/eu.esdihumboldt. hale.htmlexporter/
    7. eu.esdihumboldt.hale.instanceprovider: http://svn.esdihumboldt. eu/repo/humboldt2/trunk/hal e/eu.esdihumboldt.hale.instanceprovider/
    8. eu.esdihumboldt.hale.gmlvalidate: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/eu.esdihumboldt. hale.gmlvalidate/
    9. eu.esdihumboldt.hale.gmlwriter: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/eu.esdihumboldt. hale.gmlwriter/
    10. eu.esdihumboldt.transformer.cst: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/cst/eu.esdihumboldt.cst/
    11. eu.esdihumboldt.cst.corefunctions: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/cst/eu.esdihumboldt.cst.- corefunctions/
    12. com.onespatial.jrc.tns.oml.rif.translator : http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/com.onespatial.jrc. tns.oml.rif.translator/
    13. com.onespatial.jrc.tns.rif.exporter: http://svn.esdi-humboldt.eu/repo/humboldt2/ trunk/hale/com.onespatial.jrc. tns.rif.exporter/
In the eu.esdihumboldt.hale project, you should now be able to double-click on the HALE.product file and launch the application from there. To create a new product based on your developments, also double-click on the HALE.product file and then click the "Export an Eclipse product" button in the upper left corner.
Please note that checking out can be done anonymously, but for checking in you first need to obtain an account, which you can get from thorsten.reitz [at] igd.fraunhofer.de.
For more details see: http://community.esdi-humboldt.eu/projects/hale/wiki -> Get Started.
