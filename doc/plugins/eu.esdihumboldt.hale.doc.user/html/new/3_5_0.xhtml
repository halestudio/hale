What's new in hale»studio 3.5?

  • Deegree workspace configuration export
  • GML partitioning by spatial extent
  • Support for plugin installation via update sites

For a full list of changes, please check out the changelog on GitHub

The majority of the new features and improvements were made possible through our contracts with more than 30 customers. We'd like to thank you all for your support!

New features
Deegree feature store configuration export

A bit like the Geoserver App-Schema and Xtraserver configuration, it is now also possible to generate deegree feature store configurations from hale»studio alignments. This feature is currently not available in the command line interface.

Thanks to Landesvermessung und Geobasisinformation Brandenburg for funding this work!

GML output partitioning by spatial extent

The GML/XML writer in hale»studio already supported several partitioning modes, e.g. by Feature Type, or simply by feature count. We have now added support to partition output by spatial extent, using either a custom or a standard tile grid.

Thanks to swisstopo for funding this work!

Support for plugin installation via update sites

The way to install plugins in hale»studio has been improved: plugins can now be installed and updated from within the application via Install new software... in the Help menu. More detailed instructions on how to use this new feature is available on the Installing plugins help page.

The App-Schema plugin by Geosolutions is the first plugin that uses the new installation method. Their corresponding update site is configured as a default in hale»studio.

Thanks to GeoSolutions and Stefano Costa for contributing to this feature!
