Loading and browsing schemas

A first step to map schemas is to load them. Before you can start with the creation of a mapping you have to determine what the source and target schemas are.

Loading a schema in hale»studio provides you with the possibility to browse the schema in the Schema Explorer and to get detailed information on the defined types and their properties.

Source schema

The source schema defines the structure of the data that is to be transformed to the target schema. To import a source schema, either use File→Import→Source schema... or the Import button on the application tool bar, then select Source schema. Next you will be prompted to specify where to load the schema from, e.g. from a file or an online resource. There are also presets with predefined schema locations available. If there are multiple possible import formats for your selection, please choose the one that applies.

Target schema

The target schema defines the structure of the transformation result and is the target of the mapping. To import a target schema, either use File→Import→Target schema... or the Import button on the application tool bar, then select Target schema. Next you will be prompted to specify where to load the schema from, e.g. from a file or an online resource. There are also presets with predefined schema locations available. If there are multiple possible import formats for your selection, please choose the one that applies.

Supported schema formats

hale»studio supports the following formats for schema import:

  • XML Schema
  • GML Application Schema
  • CSV
  • Shapefile
  • Databases
