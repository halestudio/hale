Saving your work

hale»studio allows you to save your mapping, together with the configuration which schemas and source data sets are used in an alignment project. The project also includes additional settings, e.g. the list of mapping relevant types and the map styles. You can continue working on a project later on, by loading it again in the application.

To save a project, use the Save Alignment Project as... command in the File menu or the application tool bar. Recently saved or loaded projects are available in the File menu and on the welcome page for quick access, for loading another project you can use the Open Alignment Project command.

There are four different formats for projects currently supported:

  • A hale project (.hale) contains the project configuration and the alignment in a single file. The (.hale) format is deprecated and only kept for backwards compatibility.
  • A hale XML project (.halex) saves the project as an XML file, with the alignment as a separate file. This is the best choice if you use your projects with any kind of version control.
  • A hale project archive (.halez) is a ZIP archive including the project file, alignment and all local resources. Online resources can be included as well. The project archive is the best choice for sharing a project. It can be turned into a hale XML project by simply extracting the archive.
  • With a hale connect transformation project you can upload your project including the alignment and all local resources to the collaboration platform hale»connect.
